# skill 交付前检查清单

分享 / 上线前逐项核对。详细判据见 `authoring-guide.md`。

## 核心质量
- [ ] description 第三人称、具体、含关键术语
- [ ] description 同时写「做什么」和「何时用」
- [ ] description ≤1024 字符
- [ ] 命名用动名词 / 名词短语，小写+连字符，非 helper/utils/tools、非保留词
- [ ] SKILL.md 正文 ≤500 行（力求更瘦）
- [ ] 额外细则 / 长模板在单独 reference 文件
- [ ] 引用只一层深、路径存在且拼写正确
- [ ] reference `*.md` >100 行有目录（TOC）
- [ ] 自由度匹配任务脆弱性（脆弱→低自由度精确脚本）
- [ ] 无时效信息（或收进「旧模式」区）
- [ ] 全文术语一致
- [ ] 示例具体而非抽象
- [ ] 工作流步骤清晰，关键操作有验证 / 反馈循环

## 代码与脚本（如含）
- [ ] 脚本解决问题而非推卸给 Claude；错误处理明确有帮助
- [ ] 无魔法常量（每个值都有注释理由）
- [ ] 所需包在 SKILL.md 列出并确认可用
- [ ] 明确「执行」还是「作参考读」
- [ ] 路径全用正斜杠
- [ ] 高风险 / 批量操作有验证或「计划-验证-执行」

## 测试
- [ ] ≥3 个评估场景
- [ ] 用将使用的所有模型测过（Haiku / Sonnet / Opus）
- [ ] 用真实使用场景测过
- [ ] 观察过 Claude 如何浏览 skill（读文件顺序、卡点、被忽略的文件）
